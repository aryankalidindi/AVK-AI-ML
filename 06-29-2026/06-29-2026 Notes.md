**AI Response for Idea structure**

Blueprint: Siri-to-GitHub Food Ordering Bot

1. The Voice Trigger (iOS Siri Shortcut)

- **Create Shortcut**: Name it "Order Food" so Siri responds to _"Hey Siri, order food"_.
- **Action 1 (Dictate Text)**: Captures your spoken prompt (e.g., _"I want a McChicken from McDonald's"_).
- **Action 2 (Get Contents of URL)**: Sends a **POST** request containing the transcribed text payload to your automation backend server via a secure webhook.

2. The Brains (AI Intent Parsing Backend)

- **Webhook Endpoint**: A cloud service (like Make.com / Zapier) or a local machine exposed via an `ngrok` tunnel captures the incoming Siri text.
- **LLM Extraction**: Passes the text string to an AI API (OpenAI or Claude) to parse unstructured speech into a structured JSON dataset: `{"restaurant": "mcdonalds", "item": "mcchicken", "quantity": 1}`.

3. The Hands (Browser Automation)

- **Tool Selection**: Uses **Python + Playwright** equipped with a stealth plugin (`playwright-stealth`) to look like a human browser.
- **Session Persistence**: Bypasses login captchas by utilizing saved, pre-authenticated session cookies for your food delivery aggregator of choice (UberEats, DoorDash, or Grubhub).
- **Execution Loop**: Launches an invisible (headless) browser instance, navigates to the target vendor page, executes an automated keyword search for the item, and adds it to the checkout tray.

4. The Human-in-the-Loop Safeguard

- **Price Scraping**: Before confirming payment, the script pauses on the checkout screen to scrape variable data fields (service fees, delivery fees, taxes, and final totals).
- **Push Notification**: The server fires a webhook to a notification service (like **Pushover** or **Twilio**), pinging your iPhone with the live order breakdown.
- **Manual Gate**: The message includes a deep link directly to the app's checkout screen or requires a simple `"Confirm"` text reply from your phone to authorize the final payment processing.
